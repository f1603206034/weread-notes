# 第2章 线性代数

> 📌 张量
> ⏱ 2025-05-22 22:10:20

💭 什么是张量？
你可以把张量看作是多维数组 (multi-dimensional array)，它是标量、向量和矩阵概念的推广。
标量 (Scalar)：一个单独的数字。可以看作是 0 阶张量 (Rank-0 Tensor)。
例子：数字 5，房间温度 23.5。它没有维度，不需要索引就能访问。
向量 (Vector)：一列有序的数字。可以看作是 1 阶张量 (Rank-1 Tensor)。
例子：一个点的坐标 [x, y, z] 如 [2, 3, 5]。它有一个维度，需要一个索引来访问其中的元素 (例如，vector[0] 是 2)。
在机器学习中：一个数据样本的特征向量，比如一个用户的特征 [年龄, 收入, 购买次数]。
矩阵 (Matrix)：一个二维的数字网格（行和列）。可以看作是 2 阶张量 (Rank-2 Tensor)。
例子：
[[1, 2, 3],
 [4, 5, 6]]
它有两个维度（行和列），需要两个索引来访问其中的元素 (例如，matrix[0][1] 是 2)。
在机器学习中：一批数据样本，其中每一行是一个样本的特征向量；或者神经网络中一层的权重。
更高阶的张量 (Higher-Order Tensors)：
3 阶张量 (Rank-3 Tensor)：可以想象成一个“数字立方体”或一个“矩阵的数组”。
例子：一张彩色图片。它可以表示为 [高度, 宽度, 颜色通道]。例如，一张 256x256 像素的 RGB 图片就是一个 256x256x3 的3阶张量。需要三个索引 image[height_idx][width_idx][channel_idx] 来访问一个特定像素的特定颜色分量。
4 阶张量 (Rank-4 Tensor)：可以想象成一个“3阶张量的数组”。
例子：一批彩色图片 (a batch of color images)。在深度学习中，模型通常一次处理一批数据。如果批次大小为 32，图片是 256x256 RGB，那么这个张量的维度就是 [32, 256, 256, 3] ([batch_size, height, width, channels])。
以此类推... 我们可以有5阶、6阶甚至更高阶的张量。
总结一下张量的关键特性：
阶 (Rank) / 维度数量 (Number of axes/dimensions)：表示访问张量中一个元素所需要的索引数量。
标量：Rank 0
向量：Rank 1
矩阵：Rank 2
以此类推。
形状 (Shape)：一个整数元组，描述了张量每个维度的大小。
向量 [1, 2, 3] 的形状是 (3,)。
矩阵 [[1,2],[3,4],[5,6]] 的形状是 (3, 2)。
一张 256x256 RGB 图片的形状是 (256, 256, 3)。
数据类型 (Data type)：张量中所有元素的数据类型，通常是浮点数 (float32, float64) 或整数 (int32, int8)。
张量在计算机科学（特别是机器学习和深度学习）中的实际意义：
张量是现代机器学习和深度学习的基本数据结构。几乎所有的数据和模型参数都用张量来表示和操作。
统一的数据表示：
输入数据 (Input Data)：
表格数据： 可以表示为2阶张量（矩阵），[样本数, 特征数]。
时间序列数据： 如股票价格、传感器读数，可以表示为2阶张量 [时间步, 特征数] 或3阶张量 [批次大小, 时间步, 特征数]。
图像数据：
灰度图：2阶张量 [高度, 宽度] 或3阶张量 [批次大小, 高度, 宽度]。
彩色图：3阶张量 [高度, 宽度, 通道数] 或4阶张量 [批次大小, 高度, 宽度, 通道数] (例如，通道数对于RGB是3)。
视频数据： 4阶张量 [帧数, 高度, 宽度, 通道数] 或5阶张量 [批次大小, 帧数, 高度, 宽度, 通道数]。
文本数据： 经过词嵌入 (Word Embeddings) 处理后，文本可以表示为2阶张量 [序列长度, 嵌入维度] 或3阶张量 [批次大小, 序列长度, 嵌入维度]。
模型参数 (Model Parameters)：
神经网络中的权重 (Weights) 通常是2阶张量（矩阵）。例如，一个全连接层连接 n 个输入神经元和 m 个输出神经元，其权重矩阵的形状就是 [n, m] 或 [m, n] (取决于约定)。
卷积神经网络 (CNN) 中的卷积核 (Convolutional Filters/Kernels) 通常是4阶张量 [输出通道数, 输入通道数, 核高度, 核宽度]。
偏置 (Biases) 通常是1阶张量（向量）。
中间计算结果 (Intermediate Computations)：
神经网络每一层的激活值 (Activations) 都是张量。
在反向传播中计算的梯度 (Gradients) 也是张量，并且与对应参数的形状相同。
高效的数值计算：
现代深度学习框架（如 TensorFlow, PyTorch）的核心就是基于张量的运算。
这些框架提供了大量优化过的张量操作（如矩阵乘法、卷积、元素级运算等）。
这些运算可以在 GPU (图形处理器) 或 TPU (张量处理器) 上高效并行执行，极大地加速了复杂模型的训练和推理过程。如果没有张量和针对张量运算优化的硬件，深度学习的进展会慢得多。
标准化的接口和操作：
使用张量作为统一的数据结构，使得不同类型的模型和数据处理流程可以更容易地组合和交互。
张量操作（如 reshape, transpose, concatenate, slicing 等）为数据转换和操作提供了灵活且标准化的方式。
数学模型的自然表达：
许多数学和物理概念（如梯度、雅可比矩阵、海森矩阵）本身就是张量，或者可以用张量运算来方便地表示和计算。
举个简单的深度学习场景：
假设我们正在训练一个卷积神经网络 (CNN) 来识别图片中的猫。
输入： 一批猫的图片。如果批次大小是 64，每张图片是 224x224 像素的 RGB 图。那么输入数据就是一个形状为 (64, 224, 224, 3) 的 4阶张量。
第一层卷积层：
卷积核 (Weights)： 假设有 32 个 3x3 大小的卷积核，它们要处理输入图片的 3 个颜色通道。那么这组卷积核就是一个形状为 (32, 3, 3, 3) ([输出通道, 输入通道, 核高, 核宽]) 的 4阶张量。
偏置 (Biases)： 对应 32 个输出通道，会有一个形状为 (32,) 的 1阶张量 (向量)。
输出 (Activations)： 经过卷积和激活函数后，输出的特征图（feature maps）可能是一个形状为 (64, 220, 220, 32) （假设padding和stride导致尺寸略微减小）的 4阶张量。
后续层： 模型的每一层都会接收上一层的张量输出，并用自己的张量参数（权重和偏置）进行计算，然后输出新的张量，直到最后的输出层。
梯度： 在反向传播过程中，计算得到的损失函数关于每一层参数的梯度，也会是与参数形状相同的张量。例如，第一层卷积核的梯度也是一个 (32, 3, 3, 3) 的4阶张量。
总之，张量是深度学习世界的“通用语言”和“基本构件”。理解张量及其操作是理解和实现深度学习模型的基础。它们使得复杂的数据和计算能够以结构化和高效的方式进行处理。

# 第4章 数值计算

> 📌 损失函数
> ⏱ 2025-05-22 19:56:58

💭 什么是损失函数？
在机器学习中，我们通常会构建一个模型来对数据进行预测或分类。例如：
预测房价 (回归问题)
识别图片中的猫或狗 (分类问题)
翻译一句话 (序列到序列问题)
损失函数（也常被称为成本函数 Cost Function 或误差函数 Error Function）是一个用来衡量我们的模型预测结果与真实值（或标签）之间差异程度的函数。
简单来说，它告诉我们模型犯了多大的“错误”。
损失值越小，说明模型的预测结果越接近真实值，模型性能越好。
损失值越大，说明模型的预测结果与真实值偏差越大，模型性能越差。
机器学习的训练过程，本质上就是通过调整模型的参数（比如前面梯度例子中的 w 和 b），来最小化这个损失函数的值。
为什么需要损失函数？
量化模型性能： 它提供了一个具体的数值来评估模型的好坏。
指导模型优化： 它是优化算法（如梯度下降）的目标。通过计算损失函数关于模型参数的梯度，我们可以知道如何调整参数才能使损失减小，从而让模型学得更好。
一个简单的例子：预测房价 (线性回归)
假设我们想建立一个非常简单的模型来根据房子的面积 (平方米) 预测房子的价格 (万元)。
1. 数据：
我们有以下几条真实的房屋数据：
面积 (x) (平方米)	真实价格 (y_true) (万元)
50	150
70	200
100	280
2. 模型：
我们假设价格和面积之间存在线性关系，所以我们的模型可以表示为：
predicted_price (y_pred) = w * area (x) + b
其中：
w 是权重（可以理解为每平方米的价格增量）
b 是偏置项（可以理解为基础价格，或者说面积为0时的理论价格）
w 和 b 就是模型需要学习的参数。
3. 损失函数：均方误差 (Mean Squared Error, MSE)
对于回归问题，一个非常常用的损失函数是均方误差 (MSE)。它的计算方法是：
对每个数据点，计算预测值与真实值之差的平方（(y_pred - y_true)²）。平方的目的是：
确保误差是正数。
对较大的误差给予更大的惩罚。
将所有数据点的平方误差加起来。
取平均值（除以数据点的数量）。
MSE 公式： L(w, b) = (1/N) * Σ (y_pred_i - y_true_i)²
其中：
N 是数据点的数量 (在这个例子中 N=3)
Σ 表示求和 (对所有 i 从 1 到 N)
y_pred_i 是模型对第 i 个数据点的预测价格
y_true_i 是第 i 个数据点的真实价格
4. 计算损失值的过程：
假设我们的模型初始随机选择了一组参数：w = 2.0，b = 30。
现在我们用这组参数来计算损失：
数据点 1：面积 = 50, 真实价格 = 150
y_pred_1 = w * x_1 + b = 2.0 * 50 + 30 = 100 + 30 = 130 (万元)
误差平方_1 = (y_pred_1 - y_true_1)² = (130 - 150)² = (-20)² = 400
数据点 2：面积 = 70, 真实价格 = 200
y_pred_2 = w * x_2 + b = 2.0 * 70 + 30 = 140 + 30 = 170 (万元)
误差平方_2 = (y_pred_2 - y_true_2)² = (170 - 200)² = (-30)² = 900
数据点 3：面积 = 100, 真实价格 = 280
y_pred_3 = w * x_3 + b = 2.0 * 100 + 30 = 200 + 30 = 230 (万元)
误差平方_3 = (y_pred_3 - y_true_3)² = (230 - 280)² = (-50)² = 2500
计算均方误差 (MSE) - 即损失值：
Loss = (1/3) * (误差平方_1 + 误差平方_2 + 误差平方_3)
Loss = (1/3) * (400 + 900 + 2500)
Loss = (1/3) * 3800 ≈ 1266.67
这个损失值 1266.67 就告诉我们，当前参数 w=2.0, b=30 下，我们的模型预测效果有多差。
5. 如何利用损失函数进行优化 (与梯度的联系)：
现在我们有了损失值。为了改进模型，我们会使用像梯度下降这样的优化算法。
计算损失函数 L(w, b) 相对于参数 w 和 b 的梯度：∂L/∂w 和 ∂L/∂b。
这些梯度会告诉我们，为了使损失 L 减小，应该如何调整 w 和 b（增加还是减少，以及调整的幅度）。
例如，如果 ∂L/∂w 是一个正数，意味着增加 w 会导致损失增加，所以我们应该减小 w。
更新参数：
w_new = w_old - learning_rate * ∂L/∂w
b_new = b_old - learning_rate * ∂L/∂b
(learning_rate 是一个小的正数，控制每次更新的步长)
重复这个过程（计算预测值 -> 计算损失 -> 计算梯度 -> 更新参数），直到损失值足够小，或者达到预设的训练轮数。
通过这个过程，模型会逐渐学习到更好的 w 和 b 值，使得预测价格 y_pred 越来越接近真实价格 y_true，从而使损失函数的值越来越小。
总结：
损失函数衡量模型预测与真实值之间的差异。
机器学习的目标是找到一组模型参数，使得损失函数的值最小。
不同的问题类型（回归、分类等）会使用不同的损失函数。MSE是回归问题中常用的一种。其他常见的还有：
绝对平均误差 (MAE)：L = (1/N) * Σ |y_pred_i - y_true_i| (对异常值不那么敏感)
交叉熵损失 (Cross-Entropy Loss)：常用于分类问题。

# 第6章 深度前馈网络

> 📌 计算图
> ⏱ 2025-05-22 22:16:06

💭 什么是计算图？
计算图是一种用来表示和组织数学表达式或计算过程的有向无环图 (Directed Acyclic Graph, DAG)。
在这个图中：
节点 (Nodes)：代表数据（如标量、向量、矩阵、张量）或操作（如加法、乘法、函数等）。
边 (Edges)：代表数据流向，以及节点之间的函数依赖关系。数据从一个节点流向另一个节点，通常表示前一个节点是后一个节点操作的输入。
一个简单的例子：
假设我们要计算表达式：e = (a + b) * (b + 1)
我们可以将这个表达式构建成一个计算图：
输入节点 (叶子节点)： a, b, 1 (常数也可以是节点)
操作节点：
一个加法节点 c，输入是 a 和 b，输出是 c = a + b。
一个加法节点 d，输入是 b 和 1，输出是 d = b + 1。
一个乘法节点 e，输入是 c 和 d，输出是 e = c * d。
计算图的可视化表示：
    A[a (输入)] --> C{+}
    B1[b (输入)] --> C
    B2[b (输入)] --> D{+}
    ONE[1 (常量)] --> D
    C --> E{*}
    D --> E
    E --> Result[e (输出)]
    subgraph "计算过程"
        C
        D
        E
    end
方框 (A, B1, B2, ONE, Result) 代表数据节点 (变量或常量或最终结果)。
菱形 (C, D, E) 代表操作节点。
箭头表示数据流向和依赖关系。
计算图的实际意义（重点介绍计算机相关的）：
计算图在计算机科学中，尤其是在机器学习、深度学习、编译器优化等领域，具有非常重要的实际意义。
自动微分 (Automatic Differentiation) / 反向传播 (Backpropagation)：
核心应用！ 这是计算图在深度学习中最关键的作用。
前向传播 (Forward Pass)： 当我们给输入节点赋值 (例如，a=2, b=1)，数据会沿着图中的边流动，每个操作节点根据其输入计算输出，直到最终得到结果 e。
c = a + b = 2 + 1 = 3
d = b + 1 = 1 + 1 = 2
e = c * d = 3 * 2 = 6
反向传播 (Backward Pass)： 深度学习模型训练时，我们需要计算损失函数关于模型参数的梯度，以便使用梯度下降等优化算法更新参数。计算图使得这个过程系统化和自动化。
从最终的输出节点（通常是损失函数）开始，我们可以利用链式法则，沿着图反向传播梯度。
每个操作节点都知道如何根据其输出的梯度来计算其输入的梯度。
例如，要知道 e 对 a 的偏导数 ∂e/∂a：
∂e/∂c = d
∂e/∂d = c
∂c/∂a = 1
∂d/∂a = 0 (因为 d 不直接依赖于 a)
根据链式法则： ∂e/∂a = (∂e/∂c * ∂c/∂a) + (∂e/∂d * ∂d/∂a) = d * 1 + c * 0 = d
在我们的例子中，∂e/∂a = d = 2。
同样，可以计算 ∂e/∂b = ∂e/∂c * ∂c/∂b + ∂e/∂d * ∂d/∂b = d * 1 + c * 1 = d + c = 2 + 3 = 5。
深度学习框架如 TensorFlow 和 PyTorch 会隐式地为我们构建计算图。当我们定义模型和前向传播逻辑时，框架会记录下操作和依赖关系。然后，在调用 .backward() (PyTorch) 或使用 tf.GradientTape (TensorFlow) 时，框架会自动根据这个图执行反向传播，计算所有需要的梯度。这极大地简化了复杂模型的梯度计算。
依赖关系分析和调度 (Dependency Analysis and Scheduling)：
计算图明确地表示了操作之间的依赖关系。一个操作只有在其所有输入都计算完成后才能执行。
这对于并行计算非常重要。没有依赖关系的操作可以并行执行。
例如，在我们的例子中，计算 c = a + b 和 d = b + 1 可以并行进行，因为它们之间没有直接的依赖关系（都依赖于 b，但 c 的计算不依赖 d 的结果，反之亦然）。
编译器和计算引擎可以利用计算图来优化执行顺序，最大限度地利用硬件资源（如多核 CPU 或 GPU）。
中间结果的缓存和复用 (Caching and Reusing Intermediate Results)：
在计算图中，如果一个中间节点的结果被多个后续节点使用（例如，节点 b 同时是计算 c 和 d 的输入），它的值只需要计算一次，然后可以被复用。
这可以避免冗余计算，提高效率。
图优化 (Graph Optimization)：
编译器或运行时系统可以对计算图本身进行优化，例如：
常量折叠 (Constant Folding)： 如果图中有可以预先计算的常量表达式（如 2 + 3），可以在编译时直接替换为结果 5。
无效代码消除 (Dead Code Elimination)： 如果某些节点的输出没有被任何其他节点使用（即没有出边指向实际有用的计算），这些节点的计算可以被移除。
操作融合 (Operator Fusion)： 将多个连续的小操作融合成一个更大的、更高效的操作，以减少函数调用开销和内存访问。例如，将一个卷积操作、一个偏置加法操作和一个ReLU激活函数融合成一个单一的 "FusedConvBiasReLU" 操作。
模型可视化和调试 (Model Visualization and Debugging)：
计算图提供了一种直观的方式来理解复杂模型的结构和数据流。
工具如 TensorBoard (TensorFlow) 可以可视化计算图，帮助开发者检查模型结构是否符合预期，诊断问题（例如，梯度消失/爆炸时，可以追踪梯度的流向和大小）。
模型的保存和加载 (Model Saving and Loading)：
计算图的结构（以及相关的参数值）可以被序列化并保存到磁盘。之后可以重新加载这个图结构来恢复模型，用于推理或继续训练。
动态图 vs. 静态图 (Dynamic vs. Static Graphs)：
静态图 (Static Graphs)： 如 TensorFlow 1.x, Theano。先定义整个计算图，然后编译，再执行。
优点： 易于全局优化，部署方便。
缺点： 不够灵活，对于控制流（如循环和条件判断）依赖于数据的模型（如某些 RNN 结构）处理起来比较麻烦，调试也相对困难（"define-and-run"）。
动态图 (Dynamic Graphs)： 如 PyTorch, TensorFlow 2.x (Eager Execution mode)。计算图是“边运行边定义”的 (define-by-run)。每次前向传播都会即时构建一个小图。
优点： 非常灵活，易于调试 (更像传统的编程)，对处理动态结构的模型更自然。
缺点： 全局优化可能更难一些，但现代框架在这方面已经做了很多改进。
